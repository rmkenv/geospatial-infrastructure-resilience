# Culvert Analysis as a Service

This project wraps the `CulvertAnalysisSystem` Python script into a web service using FastAPI and Celery. It allows users to start a comprehensive culvert analysis for a given US county and state via a REST API. The analysis runs as a background job, and the results, including an interactive map and GeoJSON data files, can be retrieved once the job is complete.

The entire service is containerized using Docker, making it easy to set up and run.

## Prerequisites

- [Docker](httpss://docs.docker.com/get-docker/)
- [Docker Compose](httpss://docs.docker.com/compose/install/)

## How to Run

1.  **Clone the repository:**
    ```bash
    git clone <repository-url>
    cd <repository-directory>
    ```

2.  **Build and start the services:**
    From the root directory (where `docker-compose.yml` is located), run:
    ```bash
    docker-compose up --build
    ```
    This command will build the Docker image for the API and the worker, and start the API server, a Celery worker, and a Redis container. The first build may take several minutes as it needs to install all the geospatial dependencies.

    The API will be available at `http://localhost:8000`. You can view the auto-generated API documentation at `http://localhost:8000/docs`.

## How to Use the API

You can interact with the API using any HTTP client, such as `curl` or Postman.

### 1. Start an Analysis

Send a `POST` request to the `/analysis` endpoint with the county and state.

**Example:** Start an analysis for Albemarle, Virginia.

```bash
curl -X POST "http://localhost:8000/analysis" \
-H "Content-Type: application/json" \
-d '{
  "county": "Albemarle",
  "state": "Virginia"
}'
```

The API will respond immediately with a job ID and a URL to check the status:

```json
{
  "job_id": "a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8",
  "status_url": "http://localhost:8000/analysis/a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8"
}
```

### 2. Check Job Status

Use the `job_id` from the previous step to poll the status endpoint.

**Example:**

```bash
curl "http://localhost:8000/analysis/a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8"
```

While the job is running, the response will show the progress:

```json
{
  "state": "PROGRESS",
  "status": "Collecting culvert data..."
}
```

When the job is complete, the response will contain the final result, including links to the output files:

```json
{
  "state": "SUCCESS",
  "result": {
    "status": "Complete",
    "report": {
      "timestamp": "...",
      "total_culverts": 150,
      "high_risk_culverts": 25
    },
    "outputs": {
      "map": "/analysis/a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8/map",
      "culverts": "/analysis/a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8/data/culverts.geojson"
    }
  }
}
```

### 3. Retrieve Results

Once the job is successful, you can access the generated map and data files using your browser or `curl`.

-   **View the interactive map:** Open your browser to `http://localhost:8000/analysis/{job_id}/map`
-   **Download the culverts data:** `curl -o culverts.geojson http://localhost:8000/analysis/{job_id}/data/culverts.geojson`

## Project Structure

The project is organized into a `culvert_service` directory containing the application code and Docker configuration.

```
.
├── culvert_service/
│   ├── app/
│   │   ├── __init__.py
│   │   ├── celery_app.py
│   │   ├── culvert_analysis.py
│   │   ├── main.py
│   │   └── tasks.py
│   ├── Dockerfile
│   └── environment.yml
├── docker-compose.yml
└── README.md
```

## Output Files

All output files generated by the analysis jobs are stored on the host machine in the `./culvert_service/outputs/{job_id}` directory.
